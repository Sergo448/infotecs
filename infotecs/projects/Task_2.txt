Для проведения нагрузочного тестирования сетевого устройства изнакомимся с параметрами.
Общие требования к этим устройствам выглядят следующим образом:
    •UDP Throughput не менее 900 Мбит/с
    •Утилизация ЦП не выше 80%
    •Температура ЦП не выше 60 °С
    
1. Требования к методу измерения.

Метод измерения должен отслеживать уровень утилизации ЦП и температуры ЦП относительно
увеличения UDP Throughput от 900 Мбит/c до того значения, пока како-либо показатель не привысит допустимое значение.

Метод измерения должен последовательно увеличивать UDP Throughput.
Метод измерения должен заносить данные в БД об измерении утилизации и температуры ЦП.

2. Опишите метод измерения.

Алгоритм:

{
Зачем нужен тест, какие результаты ожидаются:
Тест необходим для того, чтобы определить, до какого значения можно увеличить UDP Throughput, 
пока не будет превышен один из наблюдаемых параметров.
Ожидается увидить предельное значение итилизации и температуры ЦП.

Почему испытуемый должен приложить максимум усилий для его выполнения,
возможности проверки своих сил или соревновательный мотив:
Поскольку от того, насколько надежно будет работать выбранное сетевое устройство (далее по тексту СУ),
необходимо дать возможность нескольким тестировщикам дать с ним поработать в формате соревнований.
Цель совевнований, кто сможет маквсимально эффективно нагрузить СУ, чтобы не превысить предельные значения 
утилизации или температуры.

Медленно, четко, громко, без запинок, естественным голосом каждый учавствующий в 
тестировании должен прочесть инструкцию к своим тестам с примерами.
Один из возможных сценариаев:
Берется исследуемое сетевое устройство.
Подключается к тестовому стенду.
Создается тест-сервер.
Последовательно с шагом в 1 увеличивается UDP Throughput, начиная с 900 Мбит/с.
(Увеличивается колличество сетевых запросов на тестовом стенде. Берутся замеры.
Создается виртуальная машина которая начинает посылать последовательно увелививающееся 
количество запросов на сетевое устройство. Берутся замеры.
Создается сетевой-бот, который создает сетевые интернет запросы на СУ. Берутся замеры.).
Берутся замеры температуры и утилизации ЦП.
(Создается автоматическая тестирующая машина которая в офлайн формате 
без воздействия тестировщика берет замеры)
Измерение заканчивается, когда оба параметра привысили норму.
За предельное значение UDP Throughput берется то значение, при котором было впервые
превышено какое лиюо из измеряемых значений.
Остальные данные берутся для статистики.

За время тестирования необходимо:
Провести тест графика с небольшой нагрузкой (менее 900 Мбит/с);
Провести двухсторонние измерения;
Провести односторонние измерения;
Проветси тест RFC-2544 и Y-1564;
Проверки интернет ресурсами;
Проверки с помощью утилит.


Необходимо сообщить о временном ресурсе, о правилах исправления допущенных ошибок при работе с СУ,
чего не рекомендуется делать при решении задач с СУ, к кому обращаться в случае возникновения вопросов.

Проследить за правильностью исполнения теста.

Каждый тесрирующий должен дать свои рекомендации об эксплуатапции СУ.
}

Тестовое окружение:
Сервер;
Виртуальная машина;
Тестовый стенд;
Сетевой-бот;
Автоматическая-тестирующая машина.

Условия:
Проверяется в нескольких условиях.
а) Подключение к интернет-сети;
б) Подключение к локальной-сети;
в) Без подключения к сети. Все показатели настраиваются с помощью утилит.



Определения:
"метод измерения производительности" - метод, оценивающий с помощью показателей пропускной способности,
которые отображают количество информации, переданной сетью в единицу времени и измеряются в нашем случая в Мбит/с.
"методика нагрузочного тестирования" - набор целей, условий и методов достижения получения необходимой информации
о СУ или др. устройстве, которая позволит использовать выбранное устройство с максимальной пользой, эффективностью
и безопасностью для пользователя.
